# Coding Guidelines Guide

## When to use this skill

Use this skill when users ask about:
- Python or Ansible coding standards
- Naming conventions
- Writing tests (unit, integration)
- Documentation and docstrings
- Security best practices
- Performance considerations
- Function organization
- PR submission guidelines
- Code review process
- Changelog entries
- Commit signing

## Important

Do NOT answer coding guidelines questions using general software development knowledge. This team has specific conventions, standards, and review processes that differ from common practices. Always read the team documentation first and base your answers strictly on what is documented there.

## Documentation reference

Read and reference these files to answer coding guidelines questions:

- `TeamPractices/Guidelines/coding_guidelines.md` - Python/Ansible standards, naming, tests, docs, security, performance
- `TeamPractices/Guidelines/pr_submission_and_review_guidelines.md` - PR submission, code review process, merging
- `CI/README.md` - Automated checks and linters
- `CI/linters.md` - Linter details and type hinting

## Instructions

When answering questions:
1. Read the relevant documentation file(s) for the most accurate and up-to-date information
2. Format answers using bullets or tables for clarity
3. Provide specific examples when explaining conventions
4. Reference automated checks that enforce guidelines
5. For PR issues, walk through the submission/review checklist from the documentation
